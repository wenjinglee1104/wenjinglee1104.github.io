    • 1.Redis的启动

    在RedisInsight中企图打开本地的redis时遭到了拒绝

    C:\redis>redis-cli.exe -h 127.0.0.1 -p 6379
Could not connect to Redis at 127.0.0.1:6379: 由于目标计算机积极拒绝，无法连接。

    我以为这个像是在做项目时只要连接就可以使用的服务器一样，忘了RedisInsight只是一个做连接的客户端，如果本地不打开服务的话是没办法使用的，所以仍然要定位到redis目录下执行 

    redis-server.exe redis.windows.conf

    2.字符串的拼接

    使用+字符串
    String str1 = "Hello";
String str2 = "World";
String result = str1 + " " + str2; //结果为"Hello World"
    使用concat()方法，将指定字符连接到末尾
    String str1 = "Hello";
String str2 = "World";
String result = str1.concat(" ").concat(str2);//结果为"Hello World"
    使用StringBuilder或StringBuffer
    StringBuilder sb = new StringBuilder();
sb.append("Hello");
sb.append(" ");
sb.append("World");
String result = sb.toString();//结果为"Hello World"
    使用StringJoiner，使用指定分隔符拼接多个字符串
    StringJoiner joiner = new StringJoiner(",");
joiner.add("Apple");
joiner.add("Banana");
joiner.add("Orange");        
String result = joiner.toString();//结果为"Apple,Banana,Orange"

    3.spring mvc的网址如何打开？

    原因是在一个学习项目中我使用swagger打开网址的方式怎么也打不开图示的页面，后来把页面改成doc才生效。具体哪里看出来，我现在竟然忘了，不如从头学一下。Swagger 是一个用于生成、描述和调用 RESTful 接口的 Web 服务。一般网址是

    ​ http://localhost:端口/selfpath/swagger-ui.html

    Knife4j是为Java MVC框架集成Swagger生成Api文档的增强解决方案，文档地址是

    ​ http://localhost:端口/doc.html

    参考文档是：

    https://pdai.tech/md/spring/springboot/springboot-x-interface-doc.html

    https://tobebetterjavaer.com/gongju/knife4j.html#%E6%95%B4%E5%90%88-knife4j

    5.在ApiFox里面提供参数的错误

    GET 方法将参数附加在 URL 的查询字符串中，而 POST 方法将参数放在请求的正文中。在 APIFox 中，使用 GET 方法生成的代码会将参数作为 URL 的一部分，而使用 POST 方法生成的代码会将参数作为请求体的一部分。
