面向过程和面向对象

OOP(Object Oriented Programming): 面向对象编程。这个概念我理解了多年，从很多年之前参加技术论坛别人那这个作梗当玩笑开，到今天这个理念融入生活，我才真正明白其中的哲学。

我上周五想吃牛排，就问食堂订购一批牛肉。从订购部位开始犯了难，不知道买哪里。再次就是烹饪方式，网上的教程都是黄油小锅慢煎，看起来非常考验技术。

这就是面向过程。

​

而当我走进尊贵的兰州大饭店，跟老板要一碗牛肉拉面和招牌手抓饼的时候，坐在桌子旁边等就好了。我可以坐着吃，躺着吃，也可以放盐放醋放胡椒来吃。这就是面向对象。

​

类

对象的构造在概念上包含性质和动作。就好像爱笑的振华和爱吃的小猪。所以，一个成熟的类包含字段（Field）、方法(Method)和构造方法(Constructor)

public class Person{
    //字段
    private String name;
    private Integer age;
    private Integer sex;
    
    //构造方法,因为是空的，所以可以缺省，所以常常不用动
    public Person(){
        
    }
    
    //方法
    private void eat(){
        
    }
    
     private void sleep(){
        
    }
}

new一个对象

一般写法如下，并且写在使用对象的类中

public class PersonTest{
    public static void main(String[] args){
        Person person = new Person();
    }
}

初始化对象

方法是很多的，但是常用的就这么一种。（主打一个不说废话）

@Data
public class Person{
    //字段
    private String name;
    private Integer age;
    private Integer sex;
    
    public Person(String name, Integer age, Integer sex){
        this.name = name;
        this.age = age;
        this.sex = sex;
    }
}

public class PersonTest{
    public static void main(String[] args){
        Person person = new Person("吾爱振华", 52, 1);
    }
}

关于对象

属于拔高，就是多态。父类具有某种方法，子类可以改写此类方法，实现多种形态。

public class Shape{
    public void draw(){
        System.out.println("形状");
    }
}

public class line extends Shape{
    @Override
    public void draw(){
        System.out.println("线");
    }
}

测试类这么写

public class Test {
    public static void main(String[] args) {
        Shape line = new Line();
        line.draw();
}
