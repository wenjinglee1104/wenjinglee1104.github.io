  • java: 程序包org.springframework.boot不存在的解决方法
    原因是idea配置的maven加载不到autoconfigure，将Delegate打钩(将IDE构建/运行操作委托给Maven)

  • idea创建springboot项目，Application.java不能运行问题的解决

    1. 项目配置SDK

    2. MAVEN导入失误

      打开Application.java类可以看到，好多包都没有导入成功，此时我们肯定会想到重新导入jar包，我也尝试了，但是失败了。原因很简单，切换项目后我的maven配置回到了原始配置，导致的我的本地仓库不正确，所以导入不了需要的jar包。需要重新配置maven的本地仓库，配置结束后，clean & compile。

    1. 项目的pom文件

    2. Settings和项目各方面对应起来

  • mvn无效的标记：–release

    问题是因为spring的版本不对，删掉一行即可

  • Error:java: Compilation failed: internal java compiler error 的解决方案

    该问题，一般在导入项目时会遇到。解决思路是查看项目的JDK和IDEA的配置是否一致。

    1. File –> Project Structure –> Project Settings –> Modules 或 使用快捷键 Ctrl + Alt + shift + S打开项目的 JDK配置

    2. File –> Settings –> Build, Execution, Deployment –> Compiler –> Java Compiler或 使用快捷键Ctrl + Alt + S打开配置，点击 Java Compiler查看如图所示（Target bytecode version）目标版本：查看此处JDK版本。

  • UnsupportedClassVersionError: has been compiled by a more recent version of the Java Runtime (class file version 55.0), this version (..)up to 52.0

这是查找过程最长的bug了。这句话的意思是该项目已经被JRE 55 （即Java 17）编译了，现在的版本只到 Java 8。当时我一直以为是JDK 不一致的原因，在各个地方对应着查找，是否哪个地方应该配置为Java 8。利 用对照的方法改了一遍，发现没有卵用。想起在创建项目的时候，选择了一个SpringBoot的版本，即3.0.0 以上的。于是，我把原项目delete，重新建了一个项目，这次选择的是2.7.x版本结果是这样的。
